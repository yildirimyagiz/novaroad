# 🚀 Nova Sovereign Bootstrap System

The core bootstrap engine for the Nova ecosystem, designed for 100%
sovereignty and hardware-aware performance.

## 🏗️ Architecture

- **`nova_compiler_bootstrap.zn`**: The Stage 0 → Stage 1 bootloader. Handles
  the initial lexical, syntax, and semantic verification of the compiler source
  itself.
- **`compiler_main.zn`**: The production-ready compiler orchestrator. Integrates
  the type checker, module resolver, package manager, and LLVM/MLIR backends.
- **`domain_specific_bootstrap.zn`**: Advanced bootstrap logic for domain-aware
  AI compilers (Healthcare, Finance, Space, etc.).
- **`fullstack_demo.zn`**: Unified platform demonstration showing Single File
  Component (SFC) support across Web, Desktop, and Mobile.

## ⚙️ Bootstrap Pipeline

1. **Stage 0**: Python-based pre-bootstrap compiler (`nova_stage0.py`).
2. **Stage 1**: Nova compiler compiled by Stage 0. Runs with full
   Sovereign integrity.
3. **Stage 2**: Target compiler compiled by Stage 1, achieving self-hosting
   status.

## 🚀 Usage

```bash
# Execute the bootstrap process
ln bootstrap

# Run a unified platform demo
ln run compiler/bootstrap/fullstack_demo.zn
```

## 🛡️ Sovereign Principles

- **Verified Contexts**: All native linking and machine code generation occurs

- **Hardware Awareness**: Automatic SIMD/MIMD detection and Tesla 3-6-9 harmonic
  optimization.
- **Strict Ownership**: Memory safety enforced by the Sovereign type system
  throughout the bootstrap process.
