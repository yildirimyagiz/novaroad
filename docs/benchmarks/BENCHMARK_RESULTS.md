## Compiler Optimization Analysis

### How Zero-Cost is Achieved

1. **Type Erasure**
   - Dimensions exist only during compilation
   - Runtime types are plain `f64`/`f32`
   - No runtime type information (RTTI)

2. **Constant Folding**
   - Unit conversion factors computed at compile-time
   - Inserted as constants in bytecode
   - No runtime calculations

3. **Dead Code Elimination**
   - Unused dimension information removed
   - Only actual arithmetic remains

4. **Inline Optimization**
   - Unit operations inlined automatically
   - No function call overhead

**Result:** Unit algebra code compiles to **identical bytecode** as raw arithmetic.
